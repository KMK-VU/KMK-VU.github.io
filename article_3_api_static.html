Analytics & messages API for (static) websites

Want to send me a message on my personal website without opening your email ? You can do it here !

But how is it working behind the scenes ... on a static website like a github page ? After a quick search on internet, it looks like having a hosted website with proper backend capabilities that is linked to a database is mandatory.

Let's build a simple API that can be reached by a static website to send and store messages. The benefit of having a separate API website designed for this purpose is not having to setup things again if we want to provide similar functionality to other applications.

Caveat: The sender must have access to the website that run the API. For instance, if you are behind a corporate proxy that prevents you from opening pythonanywhere sites, this solution will not work.

With this setup, we can actually try to emulate some kind of google analytics service. Here are the functionalities that I find interesting to implement for some generic trafic analytics services:

Specifications:
  • No cookies: Either there is an account/login option that indirectly ask if the visitor wants to be identified either its privacy should be respected.
  • Visit descriptions: Easier to perform if account/login functionality is implemented. Otherwise let's stick to monitoring timestamp of any visits, duration of visits and in which order pages have been visited for a single visit.
  • Device used: Mobile or desktop, to track any behavior discrepancies between mobile users and other users.

  • Device used: It seems that there are no fullproof solutions for that functionality.
